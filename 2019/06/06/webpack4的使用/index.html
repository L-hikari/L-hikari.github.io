webpack4的使用

发表于 2019-06-06 | 分类于 前端技能 | | 阅读次数: ℃

大家都知道，webpack这个东西每次更新版本的时候，向上兼容做的不太好，所以经常要踩坑，这次就说一下webpack 4.0版本我踩过的坑吧

下载安装

在3.0 的时候下载webpack只需要 npm install webpack -g就可以了，但是到了 4.0 只是下载完 webpack 是不能够运行 webpack 命令的，它会提示你没有 webpack-cli 所以你还需要在全局安装一下，执行命令

npm install webpack-cli -g

这样你的 webpack 才能够正常执行

webpack.config.js

webpack4是约定大于配置的，它会默认以你的 index.js作为入口文件出口文件也是默认为main.js也就是说你可以不用再配置文件配置也可以直接运行命令打包

配置文件这里没有什么大的变动，和之前基本一样，就是需要多添加一个 mode 属性，这个属性设置三个值

"production" | "development" | "none" 用来告诉 webpack 用最合适的方法去打包

基本设置

使用localIdentName自定义生成的类名格式，可选的参数有

  • [path] 表示样式表 相对于项目根目录 所在路径
  • [name] 表示 样式表文件名称
  • [local] 表示样式的类名定义名称
  • [hash:length] 表示32位的hash值
  • { test: /\.css$/, use: ['style-loader', 'css-loader?modules&localIdentName=[path][name]-[local]-[hash:5]'] }

使用 :local() 和 :global()

  • :local()包裹的类名，是被模块化的类名，只能通过className={cssObj.类名}`来使用

    同时，:local默认可以不写，这样，默认在样式表中定义的类名，都是被模块化的类名；

  • :global()包裹的类名，是全局生效的，不会被 css-modules 控制，定义的类名是什么，就是使用定义的类名className="类名"

  • 注意标签选择器并不支持模块化

webpack-dev-server

4.0的设置和3.0基本一致，不过就是在下载完 webpack-dev-server 后你可能要把之前所有的包重新装一下，防止之前装的包中途少了什么。重新装一下就可以了。

并且要在项目目录下安装 npm install webpack 和 npm install webpack-cli

还要注意如果你之前在 package.json 设置了 script 属性下的 dev 属性的话，要将 dev 属性改成

webpack-dev-server 这样webpack-dev-server才能生效

当然你还可以添加更多设置

webpack-dev-server --open --port 3000 --contentBase src --hot

  • --open:自动打开浏览器
  • --port: 设置端口号
  • --contentBase:让服务器打开时不要显示根目录，显示src下的内容
  • --hot: 实现热更新，这样每次打包不用全部打包只是打包更改的部分

babel

在实际开发中我们经常会用到ES6的高级语法，但是webpack对于高级的语法不能解析，这时候我们就需要一个

loader来帮我们实时编译成低级语法

安装babel

babel现在也做了更新不是之前的那些包了

npm i babel-loader @babel/core @babel/preset-env -D

如果你使用的是 react 的话还需要安装 npm i @babel/preset-react @babel/plugin-proposal-class-properties

安装完成后再根目录下创建 .babelrc文件并进行如下配置

如果你的项目中某个插件使用了严格模式下载 npm install babel-plugin-transform-remove-strict-mode

为了兼容ES7的个别语法(async await)需要下载babel-polyfill,并且在文件中引入import 'babel-polyfill'; 或者下载@babel/plugin-transform-runtime @babel/runtime 然后在.babelrc中配置即可

如果你采用了 vue-cli 构建的项目在 babel.config.js下配置

ignore: ["./src/lib/mui/js/mui.js"] 数组内是你想要忽略的文件名

配置loader

vue-loader

为了实现模块化，我们通常都是创建 .vue 文件，通过import引入，但是webpack并不能解析 .vue 文件，这时候我们又要安装 loader 了

npm i vue-loader vue-template-compiler -D

之后在配置文件中添加 {test: /\.vue$/, use: 'vue-loader'} 匹配规则

我们通过 npm 安装的 vue-loader 版本可能都是 15.0以上

这时候还要在配置文件中引入 const VueLoaderPlugin = require('vue-loader/lib/plugin'); 之后在配置文件plugins属性中添加 new VueLoaderPlugin()

样式表所需loader

如果你需要引入样式表 css scss less 等文件，需要安装相应的 loader 然后在配置文件配置匹配规则

  • npm i style-loader -D:解析样式表必须的loader
  • npm i css-loader -D:解析css文件loader
  • npm i less-loader -D:解析less文件loader，它还依赖于less，所以还要安装less
  • npm i sass-loader -D：解析sass文件loader，它依赖于node-sass，所以还要安装依赖项

文件及路径loader

npm i url-loader file-loader -D：解析url路径loader,并且安装依赖项 flie-loader

flie-loader可以解析很多文件类型，比如字体图标文件
